pijplijn


Moderne DNA-scanners maken enorme hoeveelheden ruwe data die onleesbaar zijn. Om foutjes (mutaties) in het DNA te vinden, gebruiken we drie stappen: Eerst bepaalt BWA-mem2 de juiste plek van de DNA-stukjes op het genoom. Daarna maakt Samtools de grote tekstbestanden (SAM) een stuk kleiner (BAM) om ruimte te besparen. Als laatste zoekt BCFTools in deze bestanden naar de mutaties.

Onderzoekvraag:


Welke specifieke mutaties heeft een individu ten opzichte van het menselijk referentiegenoom?
In welke genen zitten de mutaties?

Benodigdheden:


1. Software:

1.1.BWA-mem2 V2.3: https://github.com/bwa-mem2/bwa-mem2/releases

1.2.Samtools V1.23: https://www.htslib.org/download/

1.3.BCFtools V1.23: https://www.htslib.org/download/


2.Data: Raw FASTQ files (reads) en een Human Reference Genome


3.Hardware: Linux-omgeving (schoolcomputers)

Doelen & Eisen:


Opdrachtgever:

Er moet een dynamische website worden gemaakt waarbij de gebruiker een ruwe reads file kan uploaden. Voor deze file moeten dan opties gekozen kunnen worden die besproken zijn bij de tools, zoals minimale aantal nucleotides voor een match bij BWA-mem2. Hieruit komt dan een visualisatie (tabel/ grafiek) en/of de VCF-file (om de file te krijgen moet de gebruiker dit aangeven).


Eigen eisen:

1.De backend moet net in elkaar zijn gezet doormiddel van in line comments en docstrings toe te voegen. Dit is zodat anderen programmeurs het kunnen debuggen of editen.

2.Alle tools op de website die de gebruiker gebruikt moeten werken zonder problemen. Alle informatie over de verschillende opties van iedere tool moeten op de website duidelijk erop staan, doormiddel van een handleiding of een info veld. Onder andere referentiewaardes en default waardes.
